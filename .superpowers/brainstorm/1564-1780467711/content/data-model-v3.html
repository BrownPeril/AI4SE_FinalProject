设计第2节（终版）：数据模型 — 单表

MilkTeaRecord 合并消费信息，删去 ConsumeRecord

实体设计
🧋 MilkTeaRecord （唯一实体）
🔑 id Long (自增主键)
── 基本信息 ──
brandString (品牌)
drinkNameString (饮品名)
priceBigDecimal (价格)
consumeDateLocalDateTime (消费时间)
── 饮品详情 ──
sugarLevelString (糖度)
iceLevelString (冰度)
cupSizeString (杯型)
teaBaseString (茶底)
── 主观评价 ──
ratingInteger (1-5星)
commentString (点评)
wouldRecommendBoolean
createdAtLocalDateTime
updatedAtLocalDateTime
✅ 简化效果：
  • 1张表替代2张表 — 无外键、无关联查询、无 JOIN
  • price + consumeDate 从 ConsumeRecord 并入 — 一次记录就是一次消费
  • 删除 paymentMethod — 不再需要
  • API 从 10 个 CRUD 端点缩减为 5 个 — 只需 /api/milk-tea-records
  • 前端页面从 4 页缩减为 3 页 — 消费记录页面由统计页面替代
