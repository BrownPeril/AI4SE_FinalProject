设计第3节：API 设计

RESTful JSON 接口 — 前后端通信契约

API 端点总览
🧋 /api/milk-tea-records — 饮品档案接口
GET /api/milk-tea-records 获取饮品列表（分页+筛选）
POST /api/milk-tea-records 新建饮品档案
GET /api/milk-tea-records/{id} 获取单个饮品详情
PUT /api/milk-tea-records/{id} 更新饮品档案
DELETE /api/milk-tea-records/{id} 删除饮品档案
💰 /api/consume-records — 消费记录接口
GET /api/consume-records 获取消费列表（分页+筛选）
POST /api/consume-records 新建消费记录
GET /api/consume-records/{id} 获取单条消费详情
PUT /api/consume-records/{id} 更新消费记录
DELETE /api/consume-records/{id} 删除消费记录
📊 /api/stats — 统计接口
GET /api/stats/overview 总览：总消费/总杯数/均价/最常喝品牌
GET /api/stats/daily-trend 每日消费趋势（近30天）
GET /api/stats/brand-distribution 品牌消费分布（饼图）
GET /api/stats/weekly-summary 每周消费汇总（柱状图）

通用约定

  • 分页参数：?page=0&size=10，返回 { content: [], totalElements, totalPages, number, size }
  • 筛选参数：?brand=喜茶&minPrice=10&maxPrice=30&sugarLevel=半糖
  • 统一响应格式：{ code: 200, message: "success", data: {...} }
  • 错误响应：{ code: 404, message: "记录不存在", data: null }
  • 日期格式：yyyy-MM-dd HH:mm:ss
