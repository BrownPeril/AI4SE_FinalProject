设计第2节：数据模型

奶茶记录 — 核心实体与字段设计

实体关系图
🧋 MilkTeaRecord
🔑 id Long (自增主键)
── 基本信息 ──
brandString (品牌)
shopNameString (店名)
drinkNameString (饮品名)
priceBigDecimal (价格)
recordDateLocalDateTime (记录时间)
── 饮品详情 ──
sugarLevelString (糖度枚举)
iceLevelString (冰度枚举)
toppingsString (配料,逗号分隔)
cupSizeString (杯型枚举)
teaBaseString (茶底)
── 主观评价 ──
ratingInteger (1-5 星)
commentString (文字点评)
wouldRecommendBoolean (是否推荐)
── 健康&统计 ──
caloriesInteger (估算卡路里)
createdAtLocalDateTime
updatedAtLocalDateTime
SugarLevel 糖度
无糖 / 微糖 / 半糖 / 少糖 / 全糖
IceLevel 冰度
去冰 / 少冰 / 正常冰 / 多冰 / 温 / 热
CupSize 杯型
小杯 / 中杯 / 大杯 / 超大杯

设计说明

  • 单表设计 — 本项目只有一种核心实体，无需多表关联
  • toppings 用逗号分隔 — 避免引入关联表，简化查询（如"珍珠,椰果,仙草"）
  • 枚举用 String 存储 — 后端 Java Enum 验证，数据库存字符串便于阅读
  • calories 手动填写 — 用户自行估算，不做自动计算（避免过度设计）
  • recordDate 与 createdAt 分离 — recordDate 是"何时喝的"，createdAt 是"何时记录的"
